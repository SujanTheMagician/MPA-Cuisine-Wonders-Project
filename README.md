# MPA Cuisine Wonders (Foodhub Restaurant Website)

Welcome to the MPA Cuisine Wonders repository. This project was built using a structured, role-driven approach to ensure clear focus, maintainable code, and a highly polished core user experience.

## Step 1: The Role – Frontend Developer
This project was approached strictly from a **Frontend Developer** perspective. By establishing this role from day one, the scope of the project remained crystal clear: deliver a responsive, accessible, and visually engaging user interface for restaurant patrons.

**The Frontend Toolkit:**
* **HTML5 (52.1%):** Structured the semantic foundation of the website to ensure accessibility and clear content hierarchy.
* **CSS3 (45.1%):** Implemented modern, responsive layouts using Flexbox and Grid, ensuring the site looks great on everything from mobile phones to ultra-wide monitors.
* **JavaScript (2.8%):** Used lightweight, vanilla JS for essential interactive elements, avoiding the bloat of heavy frameworks for a purely UI-focused build.

## Step 2: The Problem Statement
> *"Food enthusiasts need a fast, visually clean, and intuitive platform to browse local restaurant menus and book tables without fighting through a slow or cluttered interface."*

This single sentence is the anchor of the project. Every feature, layout decision, and line of code traces back to solving this exact pain point. If a feature didn't help a user quickly find what they were craving—whether they are looking for gourmet dishes or trying to locate local street food favorites like Burmese-style Atho and Egg Bhejo—it was excluded from the scope.

## Step 3: Contribution Boundary
To maintain strict accountability and clarity on what was built, my explicit contributions to this project are clearly defined. Drawing on my background providing freelance JavaScript services and engineering responsive web interfaces, my boundaries for this repository include:

* **UI Architecture:** Designed and coded the complete visual structure of the Foodhub website.
* **Responsive Execution:** Authored all custom CSS, explicitly owning the cross-device compatibility and styling.
* **Interactivity:** Implemented the core JavaScript logic needed for menu toggling and navigation.
* *(What falls outside the boundary: I intentionally did not build a backend database or API for this iteration, keeping the focus 100% on frontend execution and user experience).*

## Step 4: The Core First
The foundation of this project is the **Menu Display and Navigation Flow**. 

Instead of getting distracted by secondary polish or unnecessary features, the initial build focused entirely on the core working parts:
1.  Loading the page rapidly.
2.  Displaying the food categories clearly.
3.  Ensuring the user could navigate the site flawlessly.

Only after this main logic worked from start to finish were additional visual refinements, hover states, and design polish added. This approach ensures the project is stable, functional, and easy to explain.
